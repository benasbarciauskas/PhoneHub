PhoneHub

Mirror and automate your iPhones and Android phones from one Mac.

A native macOS app that discovers your connected devices, docks their real mirror windows into one stage — Apple's iPhone Mirroring for iOS, scrcpy for Android — and can drive any one of them toward a plain-English goal with AI automation presets.

One window, every phone

A native dashboard for your devices

PhoneHub discovers connected iPhones and Android phones, launches each device's real mirror window, and docks it into a managed stage — so you can watch one phone up close or lay several out side by side.

Multi-device discovery

Enumerate connected iPhones (iPhone Mirroring) and Android phones (adb) in one sidebar with live state, model, and connection info.

Native mirroring & docking

Open each device's real mirror window and dock it into the stage — iOS via iPhone Mirroring (AX-positioned), Android via borderless scrcpy.

Focus & video-wall layouts

Focus a single device full-stage, or tile several mirrors into a wall and watch them all at once.

AI automation presets

Name a plain-English goal and let an agent drive the focused device toward it via mirroir / androir — reading the screen, acting, and recovering from popups.

Live run log & Stop

Every preset run streams its progress into the app in real time, with a visible Stop that ends the run immediately.

Local & credential-free

PhoneHub talks only to the devices your Mac already sees, stores no account credentials, and runs entirely on your host.

AI presets

Drive a phone toward a plain-English goal

A preset is a named goal. Running it spawns a headless claude -p agent wired to the right MCP for the focused device — mirroir for iOS, androir for Android — that reads the screen and acts until the goal is met.

How a preset runs

  1. Read the screen — describe the current UI (real element bounds and labels).
  2. Decide the next action toward the goal.
  3. Act — tap, swipe, type, or launch — and handle popups, ads, and dead ends.
  4. Repeat until the goal is met — streaming every step into the live log.
Preset · iOS
"Open Settings, go to Wi-Fi, and tell me which network is connected."
launch Settings
describe_screen → "Wi-Fi"
tap (188, 412)
describe_screen → connected: "home-5G"
■ Stop
Presets are scoped to personal automation of your own devices. PhoneHub will not run goals framed around evading platform integrity systems.

Getting started

Build & run

Requires macOS 14+ with Swift / Xcode. For Android, install adb + scrcpy; for iOS docking, grant PhoneHub Accessibility. Automation deps (mirroir / androir / claude CLI) are optional.

1 · Build the app

Clone, then build + sign with a stable identity.

# clone & build
git clone https://github.com/benasbarciauskas/PhoneHub.git
cd PhoneHub
./build-app.sh
open PhoneHub.app

2 · Android tooling (optional)

Needed for Android discovery & scrcpy mirroring.

# adb + scrcpy
brew install android-platform-tools scrcpy
adb devices
iOS docking. Grant PhoneHub Accessibility in System Settings → Privacy & Security → Accessibility so it can position the iPhone Mirroring window. The stable self-signed build keeps that grant alive across rebuilds.
